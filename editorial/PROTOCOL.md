# Editorial Protocol — Immutable Rules

**Owner:** Richard  
**Date:** 2026-04-02  
**Purpose:** Define the strict process rules for CompoundVR editorial production. These rules are immutable and override any legacy documentation or memory.

---

## 1. Dispatch Rule (ABSOLUTE)

**Richard NEVER writes articles or creates images.**

- **Writing:** Ian owns all article drafts, research, and article angle decisions
- **Images:** Maya owns all hero images and visual assets
- **Editing/Publishing:** Richard owns briefs, review, and final publication

If Ian or Maya delegate fails:
1. Retry the delegate_task
2. Simplify context and retry
3. **ESCALATE TO JEREMY** — Richard does NOT take over

**Never substitute another agent's work without explicit Jeremy approval.**

Each agent has different tuning, personality, and skills. Richard is not qualified to do Ian or Maya's jobs.

---

## 2. Briefs Are Lightweight

**Ian owns research and article angle.**

Richard's briefs contain:
- Game name, year, VR route type
- Minimal context and viability check
- Research focus areas (optional)
- Output path

Richard's briefs do NOT contain:
- Comprehensive research notes
- Platform lists or detailed specs
- Editorial angle or thesis
- Influencing context that boxes Ian in

For older games: Richard already has 90% of context in training data. Briefs should only note fluid mod status (check Discord/Reddit for current state).

---

## 3. No Placeholder Heroes

**Never publish with a placeholder hero image.**

If hero generation fails:
1. Fix the tooling (API keys, environment)
2. Escalate to Jeremy if tooling cannot be fixed quickly
3. **Publication blocks until proper hero is generated**

Placeholder heroes accumulate technical debt and violate quality standards.

---

## 4. Publication Gate

Before publication, verify:
- [ ] Article approved by Richard after review
- [ ] Hero image generated by Maya (not placeholder)
- [ ] YAML frontmatter complete
- [ ] Queue updated (marked ✅ DONE, added to Published Games)
- [ ] Commit message clear
