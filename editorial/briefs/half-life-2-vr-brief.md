## Research Phase (Before Writing)

### 1. Community & Narrative Sources
Beyond technical docs:
- **Reddit threads** — search `r/virtualreality`, `r/half-life2`, `r/moddedgames` for long-form discussions about the mod experience (comfort, immersion, bugs)
- **YouTube transcripts** — pull from walkthroughs, let's plays, and review videos; note where creators describe sensory moments (jump scares in City 17, tension during Alyx’s intro)
- **Flat2VR Discord / developer Discord** — look for early adopter feedback, known issues, community sentiment about polish vs promise
- **Online research** — Google/Scholar/forums for “Half-Life 2 VR mod review” style pieces to see what resonates

### 2. Technical Documentation (Testing)
Test and document:
- Minimum/known-good hardware requirements (headset + PC specs)
- Install friction level (file replacement vs installer vs mod manager)
- Launch args needed (track which work, which don't)
- OS/DRM compatibility (Steam VR, driver version)
- Episode-by-episode completeness

### 3. Sensory Experience Narrative
Write about how playing feels:
- **Situations that carry weight** — e.g., “You’re walking through City 17 at dusk and the ambient hum of the street sounds is so convincing you forget where your headset ends and the world begins.”
- **Disorientation vs immersion** — “The first episode’s train scene throws you off because the motion feels real, not floaty.”
- **Audio/visual fidelity** — Compare original 2D textures to VR depth cues; note if sound design breaks immersion (e.g., Alyx sighs too close)

### 4. Testing Protocol
Test each episode on this setup before reporting:
```
Headset: [test your hardware here or state "multiple headset profiles"]
PC: CPU/GPU/RAM (document exact specs)
OS: Windows version
Driver: SteamVR/runtime version
Launcher: Dedicated launcher vs direct Steam launch
```

Acceptance criteria per episode:
- Triggers work? ✅/❌
- Movement feels natural or floaty/weird? ✅/⚠️
- Interactions register correctly (grab, push, toggle) ✅/❌
- Performance issues (stuttering, frame drops)? 📊
- Comfort issues (disorientation, motion sickness triggers)? ⚠️
- Known bugs (list them with frequency)

**If a claim comes from community chatter instead of direct testing, label it clearly as hearsay.**

## Article Structure

### 1. Title & Lead
- Include mod name, Steam page link, developer attribution
- Lead: One-sentence verdict + key friction point (e.g., "The Half-Life 2 VR mod delivers a faithful conversion but requires manual file management and launch args.")

### 2. Hardware & Setup
- Tested on which setups? State clearly.
- Installation walkthrough (with screenshots of critical steps)
- Known conflicts, driver issues, or launcher quirks

### 3. Gameplay Experience
- Episode-by-episode coverage: what's there, what's missing
- Controls/movement quality assessment
- Visual/audio fidelity vs original (if applicable)
- Comfort notes (disorientation risk, recommended usage duration)

### 4. Pros & Cons
- Specific pros (faithful conversion, immersive world, etc.)
- Specific cons (manual install, no launcher, performance hiccups, etc.)

### 5. Verdict
Clear publishability judgment:
- ✅ Stable and useful for average user → confident recommendation
- ⚠️ Needs caveats or known friction → early release label with clear warnings
- ❌ Not ready → kill or defer (document why)

### 6. References & Attribution
- Credit mod developer(s), contributors, tested hardware sources
- Links to Steam Workshop, Discord, GitHub
- Methodology note: "Tested on [specifications]"
